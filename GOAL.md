### A. Charting (`captain add`)

Setup: `mkdir -p src && echo "export const x = 1" > src/a.ts`

- **A1** — `captain add "Where is X?" "In src/a.ts" --files src/a.ts` exits 0, prints `charted <id>`, and creates `.captain/entries/<id>.md`. The id is the first 8 hex chars of `printf '%s' "Where is X?" | shasum -a 256 | cut -c1-8`.
- **A2** — The entry is plain markdown with frontmatter containing `captain: 1`, `id`, an ISO 8601 `at`, and a `files:` map from `src/a.ts` to the 64-char sha256 from `shasum -a 256 src/a.ts | cut -d' ' -f1`; after the frontmatter it has exactly one `# Where is X?` heading and the answer body.
- **A3** — `.captain/map.json` is valid JSON with a row for `src/a.ts` whose `hash` equals the current sha256 and whose `entries` array contains the id. `.captain/map.jsonl` must not exist.
- **A4** — Adding with a nonexistent or non-regular file (`--files src/nope.ts`) exits nonzero and writes no entry, map row, or partial `.captain/entries/*.md` file for that question.
- **A5** — Re-adding the same question with a different answer replaces the entry (delete + add): exactly one entry file exists for that id, and its body carries the new answer.
- **A6** — From a subdirectory (`cd src && captain add ... --files a.ts`), the stored path is still relative to the project root (`src/a.ts`). `rg -n '(/Users/|/home/|/tmp/|/private/)' .captain/` finds no absolute paths anywhere in `.captain/`.
- **A7** — `captain add` works even when `captain init` was not run first: it writes the entry/map, runs qmd update if qmd is available, and emits a stderr warning rather than failing just because init was skipped.

### P. Cache-busting (`captain prune`, `captain bust`, `captain delete`)

- **P1** — With files unchanged, `captain prune` exits 0, prints nothing, and deletes nothing.
- **P2** — After `echo change >> src/a.ts`, `captain prune` deletes every stale entry file referencing it and removes the `src/a.ts` row from `.captain/map.json`; it prints `pruned N stale entries` only when `N > 0`.
- **P3** — Same for a deleted file: `rm src/a.ts` → prune removes its entry files and map rows.
- **P4** — `captain bust src/a.ts` removes every entry referencing that file, updates `.captain/map.json`, runs `qmd update`, and exits 0. Running it again with zero matching entries still exits 0.
- **P5** — `captain delete <id>` removes exactly that entry and its map references; deleting an unknown id exits 1.
- **P6** — If `.captain/map.json` is corrupted or deleted while valid entry files remain, the next command that needs the map rebuilds it from entry frontmatter instead of losing entries.

### K. Ask/recall (`captain ask`)

Use a fresh `$WORK`: `captain init --no-embedding`, then chart `src/a.ts` with a question like "where is the billing webhook routed?" and an answer that names `src/a.ts`.

- **K1** — `captain ask "billing webhook route"` exits 0 and returns the matched entry using BM25 (`qmd search -c captain -n 5 --format json` under the hood): output includes the stored question, the answer body, `src/a.ts`, and a score rendered as a percent.
- **K2** — `captain ask` hash-verifies before answering: after modifying the backing file, the same ask does not return the stale answer, the entry file is gone, and `.captain/map.json` no longer references the file.
- **K3** — With an empty chart, `captain ask "anything"` exits 0 and prints `No charted answer. Explore, then chart what you find:` plus a usable `captain add` template.
- **K4** — On this machine with `embeddinggemma` cached, a fresh project initialized with `captain init --embedding` has `.captain/config.json` set to `{"embedding":true}` and one `captain ask` succeeds through hybrid search (`qmd query -c captain -n 5 --format json`). Expect the first run to take minutes if qmd downloads the rest of the query models.

- **I1** — `captain init --no-embedding` is idempotent and creates `.captain/entries/`, `.captain/config.json` with `{"embedding":false}`, `.qmd/`, and a `.gitignore` line containing exactly `.qmd/`.
- **I2** — qmd is project-local and registered: `qmd collection list` run with cwd set to the project shows a `captain` collection pointing at `<root>/.captain/entries`, and qmd context registration exists for captain-hook.
- **I3** — `.claude/settings.json` is valid JSON wiring a SessionStart hook running `captain context` and a Stop hook running `captain nudge` (per Claude Code hooks schema: `hooks.<Event>[].hooks[].command`).
