- **C1** `capn chart "<q>" "<a>" --files <f>` exits 0 and prints `charted <id>` where `<id>` = first 8 hex of sha256(question). `.capn/entries/<id>.md` has frontmatter `capn: 1`, the id, an ISO `at`, and a `files:` map of root-relative POSIX paths â†’ sha256 hashes matching the actual files. - **C2** `.capn/map.json` maps each backing file â†’ `{hash, entries:[ids]}` consistent with entry frontmatter.
- **C3** Re-charting the same question replaces the entry (same id, new content), never duplicates.

## P â€” Cache-busting

- **P1** `capn prune` with nothing stale is silent and exits 0.

- **P4** Deleting or corrupting `map.json` is self-healing: the next command rebuilds it from entry frontmatter.
